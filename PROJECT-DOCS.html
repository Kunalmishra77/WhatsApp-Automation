Project Overview & Architecture

Agentix is a multi-tenant SaaS platform that lets businesses manage WhatsApp conversations, run bulk campaigns, build AI chatbots, and track CRM pipelines — all through a single web dashboard.

Tech Stack

⚡

Frontend

Next.js 15 (App Router), React, TailwindCSS, shadcn/ui, TanStack Query

🗄️

Database

Supabase (PostgreSQL) — Row Level Security, real-time subscriptions, storage

🤖

AI

OpenAI GPT-4o for auto-replies, intent detection, lead scoring, sentiment

💬

Messaging

WhatsApp Cloud API (Meta Graph API v19.0) — templates, media, webhooks

💳

Payments

Razorpay — subscription plans, Pro & Enterprise tiers

📧

Email

Resend (transactional) + Gmail SMTP (workspace-configured)

🚀

Hosting

Vercel — serverless API routes, Edge middleware, cron jobs

🔴

Rate Limiting

Upstash Redis — sliding window limits for API, auto-reply, WA outbound

High-Level Architecture Flow

WhatsApp User → Meta Webhook POST /api/webhooks/whatsapp → Message Handler
                                                      ↓
CSAT Check → Flow Engine (chatbot?) → Keyword Rules → AI Auto-Reply
                                                      ↓
Supabase DB (messages, conversations, contacts) → Dashboard UI (real-time)

Authentication & Multi-Workspace

How it works Supabase Auth handles login/signup. Every user belongs to one or more workspaces. A workspace = one WhatsApp Business Account (WABA). Users switch workspaces from the sidebar.

Auth Flow

User signs up/in → Supabase session (JWT) → middleware.ts checks session → workspace_members lookup

Platform Admin? → yes → /admin dashboard     Workspace member? → yes → /[workspace] dashboard

Roles

RolePermissions
platform_adminManage all workspaces, users, plans, billing — full access
ownerFull workspace control, billing, team management
adminManage contacts, campaigns, flows, templates
agentView & reply to conversations only

Key Files

middleware.ts — session check + workspace redirect on every request

lib/authz.ts — requireWorkspacePermission(workspaceId, action) — called at every API route

store/workspace.store.ts — Zustand store keeps active workspace in memory

Section 5

Campaigns (Bulk Messaging)

Campaigns send an approved WhatsApp template to a selected audience — all contacts, by tag, or by phone prefix. Can be sent immediately or scheduled for later.

Campaign Creation Wizard — Steps

① Name → ② Choose Template → ③ Audience → ④ Schedule (optional) → ⑤ Launch / Schedule

Campaign Execution Flow

POST /api/campaigns/[id]/run

≤50 contacts? → yes → executeCampaign() — synchronous

→ no → Enqueue to campaign_queue → Cron: process-campaign-queue

For each contact: → sanitize phone → WA API send → save recipient row

Campaign Status Lifecycle

📅

draft

Created but not yet sent or scheduled

🕐

scheduled

Will auto-run at scheduled_at time (cron checks every 15 min)

⚙️

running

Currently sending to recipients

✅

completed

All sends attempted — stats finalized

Analytics Per Campaign

MetricHow Tracked
Total recipientsSet at campaign run time from contacts query
SentWA API success responses counted in executor
DeliveredMeta webhook status=delivered → campaign_recipients updated
ReadMeta webhook status=read → campaign_recipients updated
FailedWA API error OR Meta webhook status=failed
RepliedInbound message from recipient within 24h of campaign

A/B Testing

A/B Campaign Split Create two campaign variants (Template A vs Template B) targeting the same audience. Contacts are split 50/50. Stats shown separately to compare performance.

Section 7

Chatbot Flows & Automations

Flows are visual drag-and-drop chatbot builders. A flow is a directed graph of nodes connected by edges. When a customer triggers a flow, the flow engine walks the graph, sending messages and waiting for replies.

Node Types

NodeWhat it does
🟢 StartEntry point — triggered by keyword or event
💬 MessageSends a text or media message to the user
❓ QuestionSends a message and waits for the user's reply
🔀 ConditionBranches based on user input or contact field value
⏰ WaitPauses flow for N hours/days (cron resumes)
🏷️ TagAdds/removes a tag on the contact
👤 AssignAssigns conversation to an agent
🔚 EndTerminates the flow session

Flow Execution

Trigger: keyword match / button click / enrollment

Create flow_session (status=active, current_node=start)

executeNode() → send message → follow edges

Question/Condition node? → wait for next inbound message

Wait node? → time-triggers cron resumes after delay

End node → session marked completed

Key Files

lib/flow-engine.ts — executeFlow(), executeNode(), findNextNode()

app/api/cron/time-triggers/route.ts — resumes Wait nodes every hour

app/api/cron/cleanup-flow-sessions/route.ts — archives stale sessions nightly

modules/flows/ — React Flow-based visual editor

Section 8

Follow-up Sequences

Sequences are drip-message automation. A contact is enrolled in a sequence and receives pre-written messages at defined time intervals (e.g., after 1 day, 3 days, 7 days).

How Sequences Work

POST /api/sequences/[id]/enroll → Check: already enrolled? → 409

Create contact_sequences row (step=0, next_send_at = now + delay_hours)

Cron fires → finds rows where next_send_at ≤ now & status=active

Send WA text message → advance step → set next next_send_at

Last step sent → status = completed
Duplicate Enrollment Protection The system now returns HTTP 409 if a contact is already actively enrolled in the same sequence.

Section 9

AI Engine (GPT-4o)

Agentix integrates OpenAI GPT-4o at multiple points in the platform. The AI is aware of the workspace's Knowledge Base and can answer product/service questions automatically.

AI Capabilities

💬

Auto-Reply

Answers customer questions 24/7 using Knowledge Base. Respects rate limit (1 reply per 30 sec per contact).

🖼️

Image Auto-Send

Detects "show me catalog" intent → searches media library by tags → sends matching images.

😊

Sentiment Analysis

Classifies each message as positive/neutral/negative. Updates contact sentiment field.

📊

Lead Scoring

AI scores contact 0–100 based on engagement patterns. Updates contact.lead_score.

🗂️

Contact Import Mapping

AI reads CSV headers and maps to contact fields automatically.

🎯

Intent Detection

Identifies if customer wants price, catalog, booking, support, etc.

AI Auto-Reply Flow

Inbound message → AI enabled for workspace? → yes → Rate limit ok?

→ yes → Build prompt: system prompt + KB articles + conversation history

GPT-4o generate reply → Send via WA API → Save to messages table

Key Files

lib/ai-reply.ts — main AI reply generator (KB-aware, history-aware)

app/api/webhooks/whatsapp/route.ts — AI invoked after CSAT/Flow/Rules pipeline

lib/rate-limit.ts — checkAutoReplyLimit(contactId) — 1 reply per 30 seconds

Section 13

Knowledge Base

The Knowledge Base (KB) is a collection of articles/FAQs that the AI reads when generating auto-replies. KB articles ground the AI in your business context — products, pricing, policies, etc.

How AI Uses the KB

Customer asks: "What's your return policy?"

AI fetches top KB articles (by relevance or recency)

Prompt = system role + KB articles + last 10 messages + new question

GPT-4o answers based on KB — not hallucinating
Best Practice Add articles for: product catalog, pricing, FAQs, shipping info, business hours, return policy, contact info. More specific articles = better AI answers.
